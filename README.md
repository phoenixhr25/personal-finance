# 个人资产收益整合表

> 用管理会计利润表的思维，把你的资产按流动性分层，算出每一层的 NPV、年化收益率和退休推算。

---

## 这是什么

一个 Jupyter Notebook 个人财务分析工具，参考**管理会计利润表结构**设计：

- 资产按流动性从上到下排列（养老金 → 储蓄险 → 基金 → A股 → 现金）
- 自动拉取 A股/基金实时行情，无需手动更新价格
- 计算每层 NPV、年化收益率、加权平均收益率
- 四期对比（基期 / 对比节点 / 当前 / 退休目标），看清资产积累轨迹
- 退休推算：按各层独立利率，告诉你退休那年资产预计是多少

**适合人群**：有社保、公积金、储蓄险、基金、A股、银行存款的在职人员，想把所有资产放在一张表里看清楚。

## 快速开始

### 1. 安装依赖

```bash
pip install akshare numpy-financial pandas matplotlib python-dateutil
```

> 或在 notebook 第一格执行，会自动安装。

### 2. 填写你的数据

打开 `income_statement_template.ipynb`，按顺序填写以下几格（每格都有 `📋` 标注）：

| 步骤 | 内容 | 数据来源 |
|---|---|---|
| 全局参数 | 折现率、退休日期、寿命预期 | 自行设定 |
| 养老保险 | 个人账户余额、预计月领 | 社保 App |
| 住房公积金 | 余额、利率 | 公积金 App |
| 储蓄险 | 现金价值、IRR、满期金额 | 保单合同 |
| 基金 | 代码、份额、成本净值 | 券商 App |
| A股/ETF | 代码、股数、成本价 | 券商 App |
| 银行存款 | 账户余额、利率 | 银行 App |
| 历史快照 | 两个过去时间点的资产估算 | 回忆/估算即可 |

> 参考 `config_example.py` 查看完整参数说明。

### 3. 运行

```bash
jupyter notebook income_statement_template.ipynb
```

或全部执行（无需打开浏览器）：

```bash
jupyter nbconvert --to notebook --execute --inplace \
  --ExecutePreprocessor.timeout=120 \
  income_statement_template.ipynb
```

## 核心概念

### 为什么按流动性排列？

管理会计利润表从固定成本到变动成本逐层剥离，这里从最难变现的资产（养老金）到最易变现（现金）逐层展示，帮助你理解**真实可动用资产结构**。

### NPV 是什么？

Net Present Value（净现值）。用折现率把未来的钱折算到今天的价值。养老金的 NPV 远高于账户余额，因为它是终身年金——你会一直领到死。

### `PROJ_INVEST_RATE` vs `weighted_return`

| 指标 | 含义 | 能用于预测？ |
|---|---|---|
| `weighted_return` | 当前持仓历史年化（含浮盈） | ❌ 不能——会被高涨标的扭曲 |
| `PROJ_INVEST_RATE` | 你对未来投资收益的假设 | ✅ 能——这才是退休推算的输入 |

历史 CAGR 也不等于投资回报率——它包含了工资转入、社保缴纳等"增量资金"，放大了增长数字。

### 行情自动获取

- **A股 / 场内 ETF**：新浪财经实时行情
- **开放式基金**：天天基金盘中估值接口
- 网络问题时自动回退到买入成本，不会报错

## 常见问题

**Q：基金名称显示代码而不是名称？**  
A：行情接口访问失败，代码作为备用显示。检查网络连接，或在 `manual_nav` 字段手动填入净值。

**Q：在公司/学校网络下无法获取行情？**  
A：接口直连境内服务器（新浪/天天基金），一般不受限制。若有系统代理，notebook 已内置绕过逻辑。

**Q：如何添加港股 / 美股？**  
A：目前仅支持 A股、场内ETF、境内开放式基金。港美股可填 `manual_price` 手动录入现价。

**Q：退休推算数字怎么理解？**  
A：是按各层独立利率的机械推算，不含未来工资转入。把它理解为"如果你今天停止工作，纯靠现有资产复利到退休"的保守估算。
