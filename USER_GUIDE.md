# 个人资产收益整合表 · 使用指引

**一句话目标**：把你名下所有资产（社保、公积金、保险、基金、股票、存款）放进同一张表，用 NPV（净现值）折算成同一把尺子，帮你看清楚「现在值多少」「退休时会有多少」。

---

## 你能用它做什么

| 功能 | 说明 |
|---|---|
| 资产总览 | 一眼看到当前总资产、浮盈、加权年化收益率 |
| NPV 折现 | 把未来才能拿到的钱（养老金、保单）折算成今天的价值 |
| 退休推算 | 按你设定的投资年化，推算退休时总资产会有多少 |
| 四期对比 | 基期 → 对比节点 → 当前 → 退休，直观看资产增长轨迹 |
| 配置保存 | 导出 JSON 文件，下次打开秒恢复所有数据 |

---

## 快速开始（3 步）

2. **点击「🚀 运行计算」**

## 各模块填写说明

### 全局参数

| 参数 | 填什么 | 说明 |
|---|---|---|
| 折现率 | 建议 `0.03` | 用于 NPV 计算，代表"钱放着不动的机会成本"，一般用无风险利率 |
| 退休推算投资年化 | `0.04`~`0.07` | 仅用于退休推算，**不要**用历史年化填这里（历史浮盈会扭曲它）|
| 预计退休日期 | 你计划退休的年份 | 影响养老金开始领取时间和退休推算终点 |
| 预期寿命终止日 | 如 `2080-01-01` | 影响 NPV 和等值月均计算区间 |
| 基期起始 / 对比节点 | 过去某两个时间点 | 用于四期对比图的前两根柱子，填估算值即可 |

---

### ① 养老保险

从社保 App（随申办 / 粤省事等）查询个人账户余额。

月领金额支持两种方式：

**方式一：按城职保公式自动推算（推荐）**

开启「按城职保公式自动推算月领金额」开关后，填写以下参数：

| 参数 | 说明 |
|---|---|
| 所在城市 | 选城市后自动填入参考社平工资，也可选「其他」手动输入 |
| 社平工资年增长率 | 预期社平工资每年增长多少，建议 3%~5% |
| 预计缴费年限 | 到退休时累计缴纳城职保的年数 |
| 缴费指数 | 缴费基数 ÷ 社平工资，按社平基数缴纳填 `1.0` |
| 退休年龄 | 影响个人账户计发月数（60岁=139个月）|

推算公式：
- 基础养老金 = 退休时社平工资 × (1 + 缴费指数) / 2 × 缴费年限 × 1%
- 个人账户养老金 = 退休时账户余额 ÷ 计发月数
- 月领金额 = 两项之和

**方式二：手动填写**

关闭推算开关，直接填入你预估或从社保机构获取的月领金额。

### ② 住房公积金

从公积金 App / 网站查询：

- **当前余额**：公积金账户现有金额（个人+单位缴纳合计）
- **预计几年后动用**：如计划 5 年后买房，填 `5`；暂不确定可填 `10`

---

### ③ 储蓄险

从保险公司 App / 年度账单查询：

- **保单数量**：有几张储蓄型保险就填几张
- **现金价值**：当前退保能拿到的金额（非保额）
- **已缴保费**：截至今天累计已交的保费总额
- **年缴保费 / 还需缴年**：还在缴费期的填，已缴完填 `0`
- **满期金额 / 距满期年**：到期能拿到的金额和还剩几年
- **保单 IRR**：保险公司有时会提供，也可用公式估算：`(满期金额 / 累计保费)^(1/总年数) - 1`

---

### ④ 基金

从基金 App（天天基金 / 支付宝等）查询持仓成本：

**格式**：每行一只，用英文逗号分隔
```
代码,份额,成本净值,买入日期
018128,13843.62,1.6540,2025-03-13
000218,6652.95,3.7318,2025-10-13
```

- **代码**：6 位基金代码
- **份额**：当前持有份额
- **成本净值**：App 显示的「成本价」或「平均持仓成本」（即移动加权均价）——**直接填 App 上的数字，不要自己用历史交易记录重新计算**。多次买入、有部分卖出的情况，券商/基金 App 已按移动加权均价自动更新，重新计算反而容易出错。
- **买入日期**：首次建仓日（用于估算持有年限，近似值即可）

> 实时净值会自动从天天基金拉取，无需手填。

### ⑤ A 股 / ETF

从券商 App 查询持仓：

**格式**：每行一只
```
代码,股数,成本价,买入日期
600036,1700,33.08,2024-01-01
159146,6400,1.104,2024-01-01
```

- **成本价**：券商显示的「移动加权均价」——**直接填 App 上的数字，不要自己用历史交易记录重新计算**。无论买了几次、卖过几次，券商 App 已按移动加权均价持续更新，这个系统不做交易记录追踪。
- **买入日期**：首次建仓日近似值即可

> 实时价格会自动从新浪财经拉取。

### ⑥ 银行存款

**格式**：每行一笔
```
银行名称,类型,余额,年化利率
工商银行,活期,50000,0.002
招商银行,定期,100000,0.020
```

- **类型**：活期 / 定期（仅作标签，不影响计算）
- **年化利率**：活期约 `0.002`，定期根据期限在 `0.015`~`0.025` 之间

### 历史快照

用于绘制四期对比图的前两个时间点，**估算值即可，不影响核心计算**：

- 将各类资产总额按「养老+公积金 / 储蓄险 / 投资 / 现金」四类分别填入
- 如果没有记录，按印象估算一个数量级即可

## 结果解读

### KPI 卡片（顶部五格）

| 指标 | 含义 | 注意 |
|---|---|---|
| 当前总资产 | 所有资产市值之和 | — |
| 浮盈 | 市值 - 成本 | 含未实现盈亏 |
| 加权年化 | 成本加权的历史年化收益率 | **诊断用，不可用于未来预测**（受持有期和历史浮盈影响） |
| NPV | 所有资产折算到今天的净现值 | 考虑了未来才能拿到的钱（养老金、保单） |
| 等值月均 | NPV 均摊到每个月的金额 | 可理解为"这些资产相当于每月给你多少钱" |

### 整合报表

按流动性分四层展示：
- **① 社会保障层**：养老、公积金（流动性最低）
- **② 保险保障层**：储蓄险
- **③ 投资层**：基金、股票
- **④ 现金等价物层**：存款（流动性最高）

## 三类增长数字的来源与区别

这是最容易混淆的部分，务必区分清楚：

### ① 历史阶段涨幅（柱状图上的 % 标注）

**计算方式**：`(本期总资产 - 上期总资产) / 上期总资产`

**⚠️ 重要限制**：这个涨幅**不等于投资回报率**。总资产的增长来源包括：
- 工资结余转入
- 单位+个人社保/公积金缴入
- 投资本身的涨跌

三者混在一起，**无法拆分**。所以历史阶段涨幅只能看趋势，不能用来评估你的投资能力。

---

### ② 加权年化收益率（KPI 卡片）

**计算方式**：以每项资产的**成本**为权重，加权汇总各资产的年化收益率。

单项年化收益率 = `(当前市值 / 成本) ^ (1 / 持有年数) - 1`

**⚠️ 重要限制**：这个数字同样**只做诊断，不可用于预测**，原因：
- 持有时间短的资产，年化会被放大（持有 3 个月涨 10% → 年化约 46%）
- 历史浮盈已经反映在当前市值里，用它预测未来等于「假设过去的高收益会永远持续」

---

### ③ 退休推算（柱状图最右侧）

**计算方式**：对各类资产分别做**复利推算**至退休年份：

| 资产类型 | 使用的增长率 |
|---|---|
| 养老 + 公积金 | 固定 5.5%（接近社保记账利率）|
| 储蓄险 | 各保单 IRR 的平均值 |
| 投资（基金 + 股票）| 左侧「退休推算投资年化」（你手动设定）|
| 现金存款 | 固定 2.5% |

**退休推算投资年化（`PROJ_INVEST_RATE`）建议值**：
- 保守：`0.03`~`0.04`
- 中性：`0.05`~`0.06`
- 激进：`0.07`~`0.08`

这个参数与历史加权年化**完全解耦**——你可以根据自己对未来市场的判断独立设定，不受历史数据干扰。推算结果不含未来工资转入，是「停止工作后纯靠现有资产增值」的保守估算。

## 配置保存与恢复

**每次更新持仓后**：
1. 填写或修改数据
2. 点「💾 导出配置」→ 下载 `finance_config.json` 保存到本地

**下次打开 app**：
1. 在侧边栏顶部上传 `finance_config.json`
2. 所有参数自动恢复
3. 点「🚀 运行计算」

> 建议把 `finance_config.json` 存在一个固定位置（如「文档/个人财务」），每次更新就覆盖旧文件。

## 常见问题

**Q：行情拉取失败怎么办？**
A：网络问题或 API 临时不可用。稍等几分钟刷新重试。基金净值最迟 T+1 更新，盘中显示的是估值。

**Q：加权年化为什么这么高 / 这么低？**
A：历史年化受持有时间影响很大——短期内涨了很多的资产会拉高整体数值。这个数字只做诊断，不要用它预测未来。

**Q：NPV 跟市值差距很大，正常吗？**
A：正常。NPV 把养老金这类未来现金流也折算进来了，所以通常高于当前市值；折现率越高，NPV 越低。

**Q：数据安全吗？**
A：所有数据都在你的浏览器本地处理，不会上传到任何服务器。导出的 JSON 文件也存在你本地。
